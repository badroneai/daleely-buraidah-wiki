## 2. ماذا نحتاج لإثراء المعلومات أكثر؟

1. **رفع نسبة الهاتف والإنستغرام** — السجلات التي لا يزال ينقصها هاتف أو إنستغرام (منها ~104 بدون هاتف، ~121 بدون إنستغرام).
2. **استكمال reference_url** — 75 كافيه بدون رابط خريطة؛ مع الرابط يصبح الجلب التالي من الخريطة أنجح.
3. **تحديث/استكمال ساعات العمل والتقييم** — للـ 60 بدون ساعات ولتحسين دقة التقييمات.
4. **استغلال "غير موجود" على الخريطة** — جزء من الـ 212 الذين سابقاً ظهروا "غير موجود" يمكن الحصول لهم على هاتف/إنستغرام/تيك توك عبر **بحث الويب** (نفس آلية المطاعم).

## 3. هل نطبّق عليه السكربت الذي عمل للمطاعم؟

**نعم.** نفس السكربت (`scrape-gmaps.py`) يعمل للكوفيهات بدون تغيير؛ الفرق فقط في `--sector cafes`.

### لماذا يناسب الكوفيهات؟

- **سكربت واحد لجميع القطاعات** — نفس المنطق: من لديه `reference_url` يُفتح مباشرة، والباقي بحث بالاسم + خيار **`--web-fallback`** لاستخراج هاتف/إنستغرام/تيك توك من الويب عند "غير موجود".
- **لديكم 179 كافيه برابط خريطة** — سيعطيهم السكربت أولوية فتح مباشر (أنجح وأسرع).
- **الباقي (~75)** — بحث بالاسم + إن لم يُوجد على الخريطة يُستخدم بحث الويب ويُستخرج ما أمكن.

### أوامر مقترحة للكوفيهات

```bash
# كل الكوفيهات + بحث ويب عند "غير موجود" (مثل المطاعم)
python scripts/scrape-gmaps.py --sector cafes --headless --web-fallback

# من جهاز 2 (مخرجات في outputs/device-2/)
python scripts/scrape-gmaps.py --sector cafes --device-id device-2 --headless --web-fallback

# أو عبر سكربت PowerShell (جهاز 2)
.\scripts\run-scrape-device2.ps1 -Sector cafes -WebFallback
```

**تجربة أولاً (مثلاً 20 كافيه):**

```bash
python scripts/scrape-gmaps.py --sector cafes --limit 20 --headless --web-fallback --device-id device-2
```

### خيارات إضافية حسب الهدف

| الهدف | الأمر |
|--------|--------|
| فقط من لديهم رابط خريطة (تحديث سريع للـ 179) | `--sector cafes --only-with-place-url --headless` |
| فقط الناقص هاتفاً (~104) | `--sector cafes --missing phone --headless --web-fallback` |
| استئناف بعد انقطاع | إضافة `--resume` |

## 4. ملخص التوصية

- **مستوى الجودة الحالي:** جيد في العنوان والتقييم والحي؛ الهاتف والإنستغرام هما الأضعف ويستحقان إثراء.
- **الإثراء:** تشغيل **نفس سكربت المطاعم** على قطاع الكوفيهات مع **`--web-fallback`** (واختيارياً `--device-id device-2` أو جهاز 1).
- **الخطوة العملية:** تجربة على 20 كافيه ثم تشغيل كامل للقائمة؛ بعدها دمج الملف الناتج `scrape-merge-ready-*.json` عبر `merge-scraped.py` كما تم مع المطاعم وجهاز 1.
